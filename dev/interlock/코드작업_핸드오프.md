## 목표
FSM 위반 판정 → 시리얼 → 아두이노 릴레이 차단까지 연결. 현재 `safety_console.py`의 `_on_interlock`이 **로그만 찍는 스텁**이라 실제 동작 안 함.

## 만들 파일 4개

### 1. `Demo/config.py` — 시리얼 설정 추가
```python
INTERLOCK_ENABLED = True
INTERLOCK_PORT    = "/dev/ttyACM0"   # 아두이노 USB
INTERLOCK_BAUD    = 115200
INTERLOCK_TIMEOUT = 1.0
```

### 2. `Demo/interlock.py` (신규) — pyserial 시리얼 매니저
- `class InterlockController`
  - `__init__`: 시리얼 열기 시도 → **실패해도 예외 없이 fallback**(로그만, GUI 정상). 스레드 안전(`threading.Lock`), 자동 재연결.
  - `set_interlock(engaged: bool)` ← FSM `on_interlock` 연결
  - `set_feedback(level)` ← FSM `on_feedback` 연결 (Feedback.NONE/WARNING/BLOCK)
  - 상태 → 명령 매핑: NONE+해제 → `RUN\n` / WARNING → `WARN\n` / BLOCK → `BLOCK\n`. `ACK\n` 수신 확인.
  - `close()`

### 3. `Demo/safety_console.py` — 스텁 교체 (배선)
- 현재: `_on_interlock`(약 511줄)·`_on_feedback`(약 515줄)이 로그만.
- FSM 생성부(약 256줄, `self.fsm = SafetyFSM(...)`) 근처에서 `self.interlock = InterlockController()` 생성.
- `_on_interlock(engaged)` → `self.interlock.set_interlock(engaged)` (+ 기존 로그 유지)
- `_on_feedback(level)` → `self.interlock.set_feedback(level)`
- 종료 시 `self.interlock.close()`

### 4. `Demo/console_interlock.ino` (신규, Arduino UNO R4)
- `setup()`: `Serial.begin(115200)`, 핀 D7·D6·D5·D4 `OUTPUT` + **초기 HIGH(릴레이 OFF — active LOW)**
- `loop()`: 시리얼 한 줄 파싱 → 릴레이 제어 → `ACK\n` 송신
  - `RUN`   → 녹 ON(D5=LOW), 적·황·부저 OFF
  - `WARN`  → 황 ON(D6=LOW), 녹 OFF
  - `BLOCK` → 적 ON(D7=LOW)+부저 ON(D4=LOW), 녹 OFF
- 아두이노 CLI로 컴파일·업로드(파이에서). IDE 아님.

## 핀·매핑 (결선도 §3·§4 정본)
| Arduino | 릴레이 | 부하 | active LOW |
|---|---|---|---|
| D7 | IN1 | 타워램프 적 (BLOCK) | LOW=ON |
| D6 | IN2 | 타워램프 황 (WARNING) | LOW=ON |
| D5 | IN3 | 타워램프 녹 (정상) | LOW=ON |
| D4 | IN4 | 타워램프 갈/부저 (BLOCK) | LOW=ON |

FSM 콜백(정본 `fsm.py`): `on_interlock(engaged: bool)`, `on_feedback(NONE/WARNING/BLOCK)`.

## 검증 (단계별)
1. **소프트(HW 없이)**: `interlock.py` 시리얼 미연결 시 fallback 동작 / mock·socat 가상포트로 명령 송신 로직 확인.
2. **아두이노 연결 후**: 스케치 업로드 → GUI 상태전이 → 릴레이 딸깍 + 표시 LED 확인.
3. **12V 부하 후**: 12V LED 파일럿(타워램프 대신, 고장 방지)으로 점등 확인 → 최종 타워램프.

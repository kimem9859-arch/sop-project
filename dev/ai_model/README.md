## 학습 결과 (best.pt, Colab T4)
- yolov8n(COCO) → 5클래스 전이학습(head 80→5), 100ep 설정 → best=ep38.
- **test mAP50 = 0.993 / mAP50-95 = 0.835.** 클래스별 mAP50: B1 .991 / B2 .995 / B3 .995 / **B4 .987(최약, 50-95 0.758)** / EMO .995. B3↔EMO 혼동 없음(색증강 억제 효과).
- 데이터셋: 박스 라벨 1589개(B1 320/B2 323/B3 308/B4 342/EMO 296), train 369·valid 35·test 18(증강 3x). `data.yaml nc:5 names:[B1,B2,B3,B4,EMO]`.
- `best.pt`(6.3MB): 노트북 `~/projects/데이터셋/runs/button_v1/weights/`.

## ⚠️ 정확도 리스크 — 2겹 (실추론 검증 필요)
1. **학습지표 낙관적**: train/val/test가 **같은 2영상 랜덤분할** → 프레임 닮은꼴이라 mAP 0.993은 과대평가일 수 있음. **다른 환경 새 영상 검증 필수.**
2. **양자화 열화**: GPU 미사용·캘리브 64장 → DFC 양자화 **level 0** → .hef 정밀도 추가 하락 가능(특히 B4).
→ 부족 시 GPU + 캘리브 1024+장 + 새 영상으로 재학습/재최적화 = **`console_v2`**.

## .hef — Pi 추론 규격 (과거 미탐지 3대 원인 차단, 반드시 지킬 것)
- 입력: **uint8 640×640 RGB** (float32 정규화 ❌ — 정규화는 칩 내부)
- 출력: **HailoRT NMS 결과** `[클래스5, 5, 100]` 파싱 (raw 텐서 디코딩 ❌)
- **HailoRT 4.x 필수** (DFC 3.33.1 hef 호환 / 5.x 금지)
- 클래스 순서 = 위 5클래스 표
- 산출물 `console_v1.hef`(4.4MB, 빌드명 `yolov8n.hef` 개명) — ✅ **Rpi5 repo·파이 반영 완료**(`Demo/models/console_v1.hef`, `config.HEF_MODEL_PATH` 연결, 실추론 검증). 빌드 원본은 `D:\Hailo_DFC\`.

## 통합 시 주의 (Pi)
- ✅ `Rpi5/Demo/recipe.json` 단계 라벨 = 정본 §6 PM 정비 시퀀스(클린·가스차단→펌프/퍼지→전극냉각→챔버벤트)로 **동기화 완료**. (버튼 클래스명 B1~B4·EMO 일치)

- 예전 14클래스 추론 코드를 5클래스로 적응(6주차 코드 재활용 가능).

## 핵심 교훈 (console_v2 재학습 시에도 유효)
1. **SAM3 폴리곤 금지** — YOLO 박스 학습엔 Object Detection 박스 라벨만.
2. **색 증강 금지** — 클래스를 색으로 구분하므로 hue/sat/grayscale 변형은 라벨 파괴.
3. **.hef ↔ 추론코드 3정합** — 입력형식(uint8)·NMS 위치·클래스 수 일치해야 탐지됨.
4. **버전 매칭** — DFC ↔ Model Zoo ↔ Pi HailoRT 호환(Hailo-8은 DFC 3.x 라인). 최신이 항상 옳지 않음.
- 라벨링: 가린 버튼도 amodal 박스, 25%↑ 보이거나 위치 식별되면 라벨.

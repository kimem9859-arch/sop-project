## 슬라이드 구성 (발표 시간 드는 = 7장)

| # | 슬라이드 | 핵심(요지만) | 시간 |
|---|---|---|---|
| 1 | **표지** | 프로젝트명(휴먼 에러 사전 예방 안전 콘솔)·팀·현 단계(PoC 통합 진행 중) | ~0.5분 |
| 2 | **프로젝트 개요** | 정의·배경 + **차별점·아이디어 구조 흡수**(사후 대응 → 사전 차단 +α 레이어 / 피지컬 AI 폐루프: 인식→판정→차단) | ~1.5분 |
| 3 | **역할 분담·조직도** | 8인 역할(1장, 가볍게) — *제출 양식 필수* | ~0.5분 |
| 4 | **HW/SW 선정·사양** | 선정 **근거 위주 1장**(Pi5+Hailo-8 / 글래스 ESP32-S3 / 인터락 Arduino+릴레이). 상세 스펙 = 부록 | ~1분 |
| 5 | **진행 일정 대비 실제 수행 현황** | **계획 간트 + 실제 진행 오버레이(통합)** — *일정표 양식 필수 → 여기서 충족* | ~1.5분 |
| 6 | **기능 구현·시스템 구성 현황** ⭐ | **결과 본체(비중 최대)** — 인식(console_v1 실추론·B4 미탐지→v2) / 판정(FSM 6상태) / 인터락(릴레이·타워램프+버튼·EMO 코드 완료·실연결) / 정직한 한계 | ~2.5분 |
| 7 | **남은 작업·마무리 계획** | console_v2 재학습·실물 결선 E2E·최종 발표 + **새 간트(향후 일정)** | ~1.5분 |
| — | **부록** (발표 시간 안 듦) | **재료비(BOM)**·HW 상세사양·기술 상세(FSM 상태도·인터락 결선·벤치마크) — 질의 대비 | 0 |

→ 일정 3종(계획·실제·향후) → 2장(5 현황, 7 새 간트)로 통합. 차별점은 2에 흡수. BOM·상세는 부록. **정의(2)·진행(5)·결과(6)·계획(7)에 시간 집중.**

## 제출 양식 필수 항목 처리
- **역할분담·조직도** → 슬라이드 3 (최소 1장 유지).
- **일정표(간트)** → 슬라이드 5(계획+실제 통합)·7(새 간트)로 충족.
- **재료비(BOM)** → 부록 (발표 시간 미소요, 양식 충족).

## 슬라이드 6 정직한 한계 (반드시 포함 — 과장 회피)
- console_v1 **B4 미탐지** → console_v2 재학습 확정
- **FPS 목표 30→15fps 하향**(ESP32-S3 경로 한계, MediaPipe 포함 재측정 필요)

- 인터락 **PoC = 타워램프 시각화**(실장비 차단 아님)

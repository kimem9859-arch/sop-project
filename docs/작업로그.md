# 작업로그 (세션 저널)

> **무엇을·언제·어느 세션(ID)·어느 커밋으로** 작업했는지 시간순으로 남기는 저널.
> 세션을 이어 작업하거나(어느 `session_id`를 resume할지) 어제 중단한 작업을 다시 잡을 때의 기준.
>
> **역할 경계 (drift 방지)** — 이 파일은 **프로젝트 작업**의 시간순 로그다(CC 작업 제외). 서로 다른 질문에 답하므로 아래와 공존한다:
> - `작업로그.md`(이 파일) = **프로젝트 작업**을 언제 어느 세션/커밋으로 했나 (시간순·append)
> - `docs/claude-code-작업로그.md` = **Claude Code 작업**의 시간순 로그 (CC는 이쪽)
> - `docs/claude-code-작업문서.md` = CC 인프라·작업방식·원칙 (설명서)
> - `CLAUDE.md`「현재 상태」 = 지금 어디까지 (스냅샷) · `타임라인.md` = 통합문서 변경 · 기준문서 §8·§9 = 마일스톤
> - `docs/superpowers/specs/*-design.md` = **작업 전 설계·결정 근거**(왜·트레이드오프·**거부한 대안**) — brainstorming 산출물. ※논의·미결만 / 확정 수치는 §10 / 완료 이력은 이 로그
>
> **표기**: ✅ 완료 / ⏸ 중단(재개 대기) / ▶ 다음(할 일) / 🔗 커밋
> **세션 ID = 전체 UUID로 표기**(짧은형 아님) — 그 줄의 ID를 그대로 `claude --resume`에 써서 세션을 이어갈 수 있게. 블록 제목 `## <날짜> · session <full-uuid>`.
> **기록 방식**: 사람/Claude가 "세션 마무리"(`/세션마무리`·스킬 `session-wrap`) 시 정리한 세션 블록(최신 위). **커밋은 각 블록의 🔗 커밋 줄에 적는다.** 하단 🔧 섹션은 **폐기된 기계 기록**(2026-07-17 종료·보존용).
> **훅**: `session-worklog-brief.sh`(시작 시 ⏸·▶ 브리핑+세션ID). ※SessionEnd 커밋 자동매핑 훅은 2026-07-17 폐기(경위 = CC 작업로그).

- ⏸ **중단 — 카메라·ESP32·GPIO 전부 미연결** 상태라 하드웨어가 필요한 것은 전부 대기. 코드로 할 수 있는 일은 이 세션에서 끝냈다.

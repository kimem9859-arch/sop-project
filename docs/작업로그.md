# 작업로그 (세션 저널)

> **무엇을·언제·어느 세션(ID)·어느 커밋으로** 작업했는지 시간순으로 남기는 저널.
> 세션을 이어 작업하거나(어느 `session_id`를 resume할지) 어제 중단한 작업을 다시 잡을 때의 기준.
>
> **역할 경계 (drift 방지)** — 이 파일은 *과거 이력 저널*이다. 서로 다른 질문에 답하므로 아래와 공존한다:
> - `작업로그.md`(이 파일) = **무엇을 언제 어느 세션/커밋으로** 했나 (시간순·append)
> - `CLAUDE.md`「현재 상태·다음 작업」 = **지금 어디까지 왔나** (현재 스냅샷)
> - `docs/타임라인.md` = **통합문서 문서변경** 이력만
> - 기준문서 §8·§9 = 진척 요약·마일스톤(M1~M5)
>
> **표기**: ✅ 완료 / ⏸ 중단(재개 대기) / ▶ 다음(할 일) / 🔗 커밋
> **세션 ID = 전체 UUID로 표기**(짧은형 아님) — 그 줄의 ID를 그대로 `claude --resume`에 써서 세션을 이어갈 수 있게. 블록 제목 `## <날짜> · session <full-uuid>`.
> **기록 방식**: 상단=사람/Claude가 "오늘 마무리" 시 정리한 세션 블록(최신 위). 하단 🔧 섹션=SessionEnd 훅이 세션→커밋을 자동 append.
> **훅**: `session-worklog-brief.sh`(시작 시 ⏸·▶ 브리핑+세션ID) / `session-worklog-commit.sh`(종료 시 커밋 매핑).

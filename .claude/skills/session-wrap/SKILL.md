# 세션 마무리 (session-wrap)

## 언제
사용자가 **"세션 마무리"**(또는 "세션 정리")라고 하거나, **한 작업 단위(A작업)가 논리적으로 끝난** 시점.
**하루의 끝·작업 종료가 아니다** — 한 세션 동안 여러 번 일어날 수 있다. "이제 그만"이 아니라 "이 갈래를 매듭짓고 기록한다"는 뜻.

## 절차

### 1. session_id 확보
SessionStart 브리핑 훅(`session-worklog-brief.sh`)이 주입한 **현재 session_id(전체 UUID)** 를 쓴다.
컨텍스트에 없으면 사용자에게 묻거나 브리핑 출력에서 찾는다. **짧은형 금지 — 전체 UUID**(그대로 `claude --resume`에 쓸 수 있어야 함).

### 2. 이번 세션 작업을 4분류로 정리
| 표기 | 뜻 |
|---|---|
| ✅ 완료 | 이번 세션에 끝낸 것 |
| ⏸ 중단 | 하다 만 것(재개 대기) |
| ▶ 다음 | 다음에 이어할 것 |
| 🔗 커밋 | 관련 커밋 해시(있으면) |

각 항목은 **사람이 읽고 이해할 의미 요약**으로 쓴다(기계적 커밋 나열 아님 — 그건 SessionEnd 훅이 이미 함).

### 3. 성격에 따라 라우팅
- **프로젝트 작업**(비전 모델·인터락·통합문서·발표 등) → **`docs/작업로그.md`**
- **Claude Code 인프라 작업**(훅·스킬·도구·작업방식 개선) → **`docs/claude-code-작업로그.md`**
- 한 세션에 **둘 다** 있으면 **양쪽에 각각** 해당 부분만 적는다.

### 4. 블록 형식 (기존 항목과 동일하게)
최신이 위로 오게 삽입:
```
## <YYYY-MM-DD> · session <full-uuid> (<한줄 제목>)
- ✅ ...
- ⏸ ...
- ▶ 다음: ...
- 🔗 커밋: <hash> · <hash>
```
날짜는 절대표기(상대날짜 금지). 상대 날짜가 나오면 실제 날짜로 변환.

### 5. 커밋
로그 파일 수정을 커밋한다(사양·설계 문서 → `sop-project`). 커밋 끝에:
`Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`
push는 사용자 요청 시.

## 주의
- 로그 = **시간순 저널**. CLAUDE.md「현재 상태」(스냅샷)와 역할이 다르다 — 공존·분리.
- SessionEnd 자동 커밋매핑은 프로젝트/CC 성격을 구분 못 해 `작업로그.md` 하단 🔧 섹션에 무분류로 쌓인다. **이 수동 요약이 의미 기록의 진짜 안전판.**
- 과거 저널 항목은 고치지 않는다(그때의 기록). 새 블록만 추가.

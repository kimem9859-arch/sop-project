# 세션 마무리 (session-wrap)

## 언제
사용자가 **"세션 마무리"**(또는 "세션 정리")라고 하거나, **한 작업 단위(A작업)가 논리적으로 끝난** 시점.
**하루의 끝·작업 종료가 아니다** — 한 세션 동안 여러 번 일어날 수 있다. "이제 그만"이 아니라 "이 갈래를 매듭짓고 기록한다"는 뜻.

## 절차

### 1. session_id 확보
SessionStart 브리핑 훅(`session-worklog-brief.sh`)이 주입한 **현재 session_id(전체 UUID)** 를 쓴다.
컨텍스트에 없으면 사용자에게 묻거나 브리핑 출력에서 찾는다. **짧은형 금지 — 전체 UUID**(그대로 `claude --resume`에 쓸 수 있어야 함).

### 2. 미기록 커밋 소급 점검 (이전 세션 누락 구제 — 2026-07-18 신설)
마무리 전에 **이전 세션이 마무리를 안 하고 남긴 커밋**이 있는지 점검한다(SessionEnd 자동매핑 훅은 2026-07-17 폐기 — 이 단계가 유일한 안전망):
1. **양 repo**(sop-project·Rpi5)에서 `git log`로 최근 커밋 해시를 뽑는다(범위 = 작업로그 최신 블록 날짜 이후, 최소 2주).
2. 각 해시(7자리)를 `docs/작업로그.md` + `docs/claude-code-작업로그.md`에서 grep — **어느 쪽에도 없으면 미기록 후보**.
3. 제외: 현재 세션 커밋(이번 블록에서 다룸) · 기록 행위 자체 커밋(`docs(세션마무리)`·작업로그 자동기록·훅 매핑류).
4. 미기록이 있으면 **사용자에게 확인**: "이전 세션 커밋 N건이 작업로그에 기록되지 않았습니다 — 같이 기록할까요?" (목록 제시). **묻지 않고 임의 소급 금지.**
5. 승인 시 **별도 소급 블록**으로 기록: `## <커밋 날짜> · session <이전ID> (소급 — 커밋 기반)`. 이전 세션 ID는 세션파일(`~/.claude/projects/…`)·resume 목록·커밋 시각으로 추정하고, 불확실하면 `session 미상(소급)`으로 표기. 블록 첫 줄에 **"커밋 메시지 기반 소급 — 무커밋 작업·중단 사유는 유실"** 한계를 명시한다(현 세션 블록과 섞지 말 것).

### 3. 이번 세션 작업을 4분류로 정리
| 표기 | 뜻 |
|---|---|
| ✅ 완료 | 이번 세션에 끝낸 것 |
| ⏸ 중단 | 하다 만 것(재개 대기) |
| ▶ 다음 | 다음에 이어할 것 |
| 🔗 커밋 | 관련 커밋 해시(있으면) |

각 항목은 **사람이 읽고 이해할 의미 요약**으로 쓴다(기계적 커밋 나열 아님 — 그건 SessionEnd 훅이 이미 함).

### 4. 성격에 따라 라우팅
- **프로젝트 작업**(비전 모델·인터락·통합문서·발표 등) → **`docs/작업로그.md`**
- **Claude Code 인프라 작업**(훅·스킬·도구·작업방식 개선) → **`docs/claude-code-작업로그.md`**
- 한 세션에 **둘 다** 있으면 **양쪽에 각각** 해당 부분만 적는다.

### 5. 블록 형식 (기존 항목과 동일하게)
최신이 위로 오게 삽입:
```
## <YYYY-MM-DD> · session <full-uuid> (<한줄 제목>)
- ✅ ...
- ⏸ ...
- ▶ 다음: ...
- 🔗 커밋: <hash> · <hash>
```
날짜는 절대표기(상대날짜 금지). 상대 날짜가 나오면 실제 날짜로 변환.

### 6. 커밋
로그 파일 수정을 커밋한다(사양·설계 문서 → `sop-project`). 커밋 끝에:
`Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`
push는 사용자 요청 시.

## 주의
- 로그 = **시간순 저널**. CLAUDE.md「현재 상태」(스냅샷)와 역할이 다르다 — 공존·분리.
- SessionEnd 자동 커밋매핑은 프로젝트/CC 성격을 구분 못 해 `작업로그.md` 하단 🔧 섹션에 무분류로 쌓인다. **이 수동 요약이 의미 기록의 진짜 안전판.**
- 과거 저널 항목은 고치지 않는다(그때의 기록). 새 블록만 추가.

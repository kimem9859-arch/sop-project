"세션 마무리" 의식을 수행하라.

**정의**: 하루의 끝·작업 종료가 아니라 **한 세션/작업 단위(A작업)가 논리적으로 끝난 시점**의 기록이다(하루에 여러 번 가능).

**절차 정본** = `.claude/skills/session-wrap/SKILL.md`. 그 단계를 그대로 따르라:
1. 현재 session_id(전체 UUID) 확보 — SessionStart 브리핑 훅이 주입한 값.
2. **미기록 커밋 소급 점검** — 이전 세션이 마무리 없이 남긴 커밋을 양 repo `git log` × 작업로그 대조로 찾고, 있으면 **사용자에게 "같이 기록할까요?" 확인 후** 별도 소급 블록으로(임의 소급 금지. 상세 = SKILL §2).
3. 이번 세션 작업을 4분류(✅완료/⏸중단/▶다음/🔗커밋)로 정리(의미 요약).
4. 성격 라우팅 — 프로젝트 작업 → `docs/작업로그.md`, CC 인프라 작업 → `docs/claude-code-작업로그.md`(둘 다면 양쪽에).
5. 블록 형식 `## <YYYY-MM-DD> · session <full-uuid> (<제목>)` 로 최신이 위에 오게 삽입.
6. 커밋(사양·설계 문서는 sop-project). push는 사용자 요청 시.

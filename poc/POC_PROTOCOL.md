# PoC Step 1 — 수행 프로토콜 (손 hover/lock-on 분리 검증)

> 목적: **버튼 검출 모델(YOLO) 없이**, 사전학습 MediaPipe 손 모델만으로
> "누르기 직전 손이 어느 버튼 ROI 위에 있는가(lock-on)"를 안정적으로 잡는지 검증한다.
> 카메라 고정 가정으로 **손 쪽 로직만 분리**한다. (버튼 동적검출=Step 2)

---

## 0. 측정 가설 (왜 'lock-on'인가)
실제 시스템에서 **'누름'은 하드웨어 디지털 입력**으로 들어온다. 비전의 역할은
누르기 *직전* 손이 향한 버튼을 인식해 **사전 경고**하는 것이다. 그러므로 측정 대상은
"가림 도중 추적"(가림=검출0이라 자기모순)이 아니라 **"누름 직전 lock-on ROI 정확도"**다.

> ### ⚠️ 전제 — 버튼 배선 불필요 (선 안 연결된 버튼으로 진행)
> 이 PoC는 **버튼 입력 신호(배선)가 필요 없다.** 여기서 말하는 "누름(contact)"은
> 전기 신호가 아니라 **영상을 보고 사람이 라벨로 표기하는 기준 시각**(§4의 GT `contact` 구간 `t_start`)일 뿐이다.
> - 검증 대상 = **순수 비전(손 hover/lock-on/dwell)**. 입력은 녹화 영상 + 사람 라벨뿐.
> - **하드웨어 누름 신호 경로는 PoC 범위 밖** (배선 후 trivial하게 추가, 나중에).
> - 따라서 **선이 연결 안 된 버튼만 있어도 그대로 검증 가능**하다.

## 1. 검증 질문 → 지표
| # | 질문 | 지표 (score.py) |
|---|------|------|
| Q1 | 누름 직전 어느 버튼인지 맞히나 | **lock-on 정확도** |
| Q2 | 스침을 위반(dwell)으로 오인하나 | **스침 오탐율** |
| Q3 | 조작 순서를 맞게 인식하나 | **dwell 순서 일치(+편집거리)** |
| 보조 | 전반 인식·임계 | 프레임 ROI 정확도·혼동행렬, dwell 임계 sweep |

## 2. ✅ 성공 기준 (분석 *전에* 고정 — 사후 합리화 방지)
정상 클립 기준, 아래 **3개 모두 충족 시 "손-only 접근 통과"**:
| 기준 | 임계 |
|------|------|
| lock-on 정확도 | **≥ 90%** |
| 스침 오탐율 | **≤ 10%** |
| dwell 순서 정확일치율 | **≥ 80%** (정상 클립 중) |
| (보조) 손 검출률(가림 제외) | ≥ 80% |

**판정 규칙**: 3개 충족 → Step 2(버튼 동적검출)로 진행. 하나라도 미달 →
원인 분석(손끝 환각/단일점/임계) 후 보완(다중 랜드마크·임계 튜닝) 또는 접근 재설계.

## 3. 촬영 셋업 (Step 1 = 카메라 고정)
- 버튼 4개: 서로 구분되게(색 또는 B1~B4 라벨), **왼→오**, 박스/종이/키패드에 부착. **배선 불필요**(누름은 영상 라벨로 처리, §0).
- 카메라: **삼각대/거치로 고정**(머리 안 움직임). 1인칭 화각 모사(머리 높이에서 버튼 내려다봄).
- 해상도 720p 권장, **fps 메모**. 클립당 카메라 위치 불변(흔들리면 그 클립 제외 표기).
- 누름 직후 손을 떼 다음 동작과 구분되게.

### ⭐ 재촬영 가이드 v2 (1차 PoC 실패모드 차단)
1차 클립에서 MediaPipe 손 검출이 0~100%로 들쭉날쭉 → 원인 3개를 막는다:
1. **손이 항상 화면 안에** (화면밖 실패 대응): 카메라를 **더 멀리/높게** 두어 누를 때
   **손 전체(손목~손끝)** 가 프레임에 들어오게. 버튼은 중앙·사방 여백 충분.
   손은 아래에서 진입, **가장자리서 손가락만 나오지 않게.**
2. **누르기 전 'hover 1초'** (누름=색버튼 가림 대응 + lock-on 측정 핵심):
   각 버튼마다 **접근 → 버튼 위에 손 멈춤(버튼 아직 보이게) ~1초 → 누름 → 손 떼기.**
3. **동작 분리** (구간 구분·자세 안정): 누름 사이마다 **손을 완전히 뺐다가** 다음 버튼.
   한 번에 한 동작, 천천히(모션블러 X), 조명 균일, 손목 잘리지 않게.

### 촬영 매트릭스 (`poc_data/clips/`)
| 종류 | 개수 | 내용 |
|------|------|------|
| 정상 | 5+ | B1→B2→B3→B4 순서대로 누름 |
| 위반 | 3+ | 단계 건너뜀 / 역순 |
| 스침 | 3+ | 손이 버튼 위를 지나가되 **안 누름** (오탐 측정) |
| 변형 | 가능시 | 조명·좌우손·속도 |

## 4. GT 라벨링 (구간 단위 — `poc_data/ground_truth_segments.csv`)
클립단위론 정확도 산출 불가 → **구간 단위**로 라벨.
- 컬럼: `clip, seg_id, t_start, t_end, gt_roi, action, intent, note`
- `action`: approach / **contact(=누름)** / occluded(누르며 가림) / graze(스침) / none
- `intent`: intent(의도 누름) / none
- **누름 시작 시각 = contact/occluded 구간의 `t_start`** (lock-on 기준점)
- 신뢰도: 가능하면 **2명이 라벨 → 불일치 구간 합의**(특히 graze vs intent 경계).
- 1인칭에서 누름 시점 단서: 손 정지·버튼 그림자·미세 진동.

## 5. 실행 순서 (명령)
```bash
# (1) 첫 프레임 떠서 버튼 위치 확인 → poc/rois.json 에 B1~B4 박스 좌표 입력
./poc/run.sh poc_data/clips/정상01.mp4 --dump-frame0 poc/frame0.png

# (2) 계측: 모든 클립 → out/ 에 frames.csv·events.csv + 종합표
./poc/run.sh poc_data/clips/ --rois poc/rois.json --out poc/out

# (3) 채점: frames.csv ↔ GT → 지표
.poc_venv/bin/python poc/score.py --frames poc/out --gt poc_data/ground_truth_segments.csv --sweep
```

## 6. 통제 (결과 오염 방지)
- 카메라 실제 고정 확인(첫/끝 프레임 배경 정합) — 흔들린 클립 분석 제외.
- dwell 임계는 **sweep으로 데이터에서 선택** 후 그 값으로 본 채점(1.0초는 출발점).
- 양손 등장 시 작업손 선택 로깅(`handed` 컬럼) — 엉뚱한 손 잡는지 점검.
- 가림 구간은 프레임 정확도에서 **분리 집계**(가림제외/포함 둘 다 본다).

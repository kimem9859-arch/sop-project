> 작성일: 2026-06-07

## 1. 개요

| 항목 | 내용 |
| --- | --- |
| **목적** | 버튼 검출 모델 없이, **사전학습 MediaPipe 손 모델 + 색 기반 버튼 인식**만으로 "1인칭 영상에서 작업자가 어느 버튼을 어떤 순서로 조작했는지" 비전이 인식할 수 있는지 검증 |
| **범위** | 손 인식 로직만 분리 검증(Step 1). 학습형 버튼 검출(YOLO)·실시간 하드웨어 경로는 Step 2 |
| **대상** | 버튼 2개(노랑=B1, 하양=B2), 핸드폰 1인칭 촬영(FHD 60fps) |
| **결론** | ✅ **통과** — 적절한 프레이밍에서 조작 순서를 97~100% 정확, 헛알람 0%로 인식 |

## 2. 검증 가설 · 성공 기준

**핵심 가설**: 실제 시스템에서 '누름'은 하드웨어 신호가 담당하고, **비전의 역할은 누르기 직전 "손이 어느 버튼을 향하는가"를 인식해 사전 경고**하는 것이다. 따라서 비전이 **버튼 ID와 조작 순서를 정확히 읽는지**가 핵심.

| 성공 기준 | 목표 |
| --- | --- |
| 조작 순서 일치도 | ≥ 80% |
| 스침(비누름) 오탐율 | ≤ 10% |
| 손 검출률(손이 보일 때) | 높을수록 |

---

## 3. 실험 방법

**인식 파이프라인 (매 프레임)**
```
1인칭 영상 → ① 손끝 검출(MediaPipe, 검지끝)
            → ② 버튼 위치 검출(색: 노랑/하양 → B1/B2)
            → ③ 손끝이 어느 버튼 영역(ROI) 안인가
            → ④ 같은 버튼에 일정시간 머물면(dwell) "그 버튼 조작"으로 판정
```
- **계측 도구** `roi_hover.py`: 프레임별 손끝·버튼ROI·dwell 기록
- **채점 도구** `score.py`: 검출 순서 vs 정답 순서(편집거리), 스침 오탐, 손검출률
- **갭 메우기(hysteresis)**: 누름 순간 인식이 잘게 끊기는 현상(채터)을 짧은 끊김 이어붙여 보정

---

## 4. 진행 과정 — 4차 반복 (실패→진단→개선)

> 이 PoC의 핵심은 **문제를 조기에 발견하고 원인을 데이터로 진단해 개선**한 반복 과정이다.

| 차수 | 조건 | 손 검출률 | 결과 | 진단·개선 |
| --- | --- | --- | --- | --- |
| **1차** | 핸드헬드, 손 일부 화면 밖 | 36% | 순서 절반 놓침 | 프레임 분석 → **손이 화면 밖**일 때 검출 실패 발견 |
| **2차** | 손 전체 화면에 (프레이밍 개선) | **96%** | recall 57%→**86%** | 누름 순간 **채터** 발견 → **갭 메우기 0.3s** 도입 |
| **3차** | 버튼을 양 끝에 배치 | 8~54% | 하양 누름 다수 놓침 | 진단: **버튼이 가장자리** → 누를 때 손 잘림. (색 검출은 100% 정상) |
| **4차** | **버튼 가운데로 모음** | **97~100%** | ✅ 순서 정확 일치 | 최종 검증 성공 |

**진단으로 확인된 핵심 사실**: 검출 실패의 원인은 색·조명이 아니라(색 검출 98~100%로 항상 정상) **언제나 "손이 화면 밖으로 나가는 것"**(가장자리 버튼·빠른 모션)이었다.

## 5. 최종 결과 (4차, 버튼 가운데)

| 클립 | 손 검출률 | 검출 순서 vs 정답 | 판정 |
| --- | --- | --- | --- |
| **정상** | **100%** | 13/13, **편집거리 0 (완벽 일치)** | ✅ |
| **위반** | 97% | 13/14, 편집거리 1 (첫 1개만 놓침) | ✅ |
| **스침** | 100% | **오탐(잘못된 dwell) 0개** | ✅ |

**성공 기준 대조**

| 기준 | 목표 | 결과 |
| --- | --- | --- |
| 조작 순서 일치도 | ≥ 80% | **정상 100% / 위반 93%** ✅ |
| 스침 오탐율 | ≤ 10% | **0%** ✅ |
| 손 검출률 | — | **97~100%** ✅ |

→ 정상 클립의 `노-흰-노-흰…노-노-흰-노-흰`(13단계, 중간 중복 포함) 순서를 **하나도 틀리지 않고** 인식.

## 6. 핵심 발견 · 확정 파라미터 (제품 설계에 반영)

1. **프레이밍이 결정적** — 누를 때 손 전체가 화면에 들어와야 함. → 스마트글래스 화각·콘솔 버튼 배치(중앙 집중) 설계의 **필수 조건**.
2. **갭 메우기 0.3초** — 누름 순간 인식 끊김(채터) 제거에 필수. → FSM 판정에 반영.
3. **색 기반 버튼 인식은 견고** — 검출률 98~100%, 카메라 흔들림(핸드헬드) 자동 흡수.
4. **"접근→잠깐 멈춤→누름" 동작** — 누름 순간 손이 버튼을 가려도, *멈춤(버튼 보이는) 구간*에 ID를 확정해 순서가 정확.

## 7. 한계 · 다음 단계

**한계**
- **가림 순간의 색 ID 상실**(구조적): 손가락이 색 버튼을 덮으면 색으로는 버튼 식별 불가. → 설계상 **하드웨어 누름 신호 + 접근 시 사전 인식**이 보완(본 PoC에서 작동 확인).
- 표본: 단일 작업자·핸드폰 촬영. 다양한 작업자·조명·실장비 일반화는 후속 과제.

**다음 단계 (Step 2)**
- **학습형 버튼 검출 모델(`console_v1` YOLO)**: 색 대신 형태로 인식 → 색 의존 탈피 + 부분 가림 버튼 인식.
- **실시간 경로 측정**: ESP32-S3(OV3660) 글래스 → RPi5 + Hailo-8 FPS.
- **하드웨어 누름 연동(인터록)**: 비전 사전경고 + 하드웨어 차단의 결합.
